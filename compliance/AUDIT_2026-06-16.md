# TuGymPR — Pre-Submission Compliance Audit & Fix Log (2026-06-16)

Full verification of the codebase (tech, APIs, data collected) vs. what's declared, then
fixes against Apple App Store & Google Play requirements. Bundle `com.tugympr.app`.

## Verdict

The hard architectural requirements are genuinely met: in-app account deletion truly deletes
(auth user + storage + ~60 tables), per-feature AI consent gates every OpenAI call, push has a
transactional/promotional split, email-only auth (Sign in with Apple N/A), analytics is
privacy-minimized (ATT N/A, `NSPrivacyTracking=false` correct), no ad SDKs, no hardcoded secrets,
and the live privacy policy / EULA / deletion pages are deployed and complete. What remained was
config hygiene + a few disclosure/security gaps — fixed below.

## ✅ Fixed in this pass

### iOS native config
- Removed `NSMotionUsageDescription` (App **and** Watch) — no CoreMotion code existed (auto-reject risk).
- Removed `NSLocationAlwaysAndWhenInUseUsageDescription` — code only ever requests WhenInUse; enriched the WhenInUse string to cover in-session background tracking (Guideline 2.5.4).
- Removed unused `remote-notification` from `UIBackgroundModes` (push is alert-only).
- `PrivacyInfo.xcprivacy`: removed the inaccurate **CoarseLocation "gym check-in"** entry (there is no GPS check-in) and **added Phone Number** (collected at onboarding + admin SMS).
- Reworded `NSPhotoLibraryAddUsageDescription` to match the real (share-card) save.
- `App.entitlements`: dropped the dev-only `?mode=developer` associated-domain.
- Version `1.0.1 → 1.1.0`, build `2606140003 → 2606160001` (App, Watch, Widget).
- All four plists/entitlements pass `plutil -lint`.

### Android native config
- Removed `ACTIVITY_RECOGNITION` (unused; Health Connect `READ_STEPS` doesn't need it).
- Fixed the Health Connect read/write mismatch so the manifest **exactly matches `healthSync.js`**:
  added `READ_HEIGHT`, `READ_SLEEP`, `READ_HEART_RATE_VARIABILITY`, `READ_RESTING_HEART_RATE`,
  `READ_EXERCISE` (all backed by live recovery/onboarding/cardio-import features); removed unused
  `WRITE_EXERCISE`. Manifest is well-formed (`xmllint`).
- Version `1.0.4 → 1.1.0`, versionCode `5 → 6`.

### Privacy / disclosure
- **Self-hosted Google Fonts** — eliminated the runtime IP→Google egress on app launch. 40 woff2
  files in `public/fonts/`, CSP cleaned of `fonts.googleapis.com`/`fonts.gstatic.com`, PWA
  font-caching rules removed, build green.
- **Generated-HTML fonts (print cards + emails)** — closed the residual below. The admin print
  sheet now loads self-hosted `public/fonts/print-fonts.css` (16 woff2: EB Garamond, DM Sans,
  Caveat; JetBrains Mono already shipped) instead of Google — the CSP (`style-src 'self'`;
  `font-src 'self'`) already blocked the old Google link, so this also restores the intended,
  previously-broken card typography. The designer email templates drop the Google Fonts `<link>`
  entirely (email clients can't load `@font-face` reliably, and it leaked each recipient's IP +
  open event to Google on render) and degrade to high-quality system fonts (Apple Mail → SF Pro /
  Georgia / SF Mono, Gmail / Android → Roboto, Outlook → Arial). Net result: **zero third-party
  font egress anywhere in the product.** Build green; print fonts runtime-verified (200
  same-origin, correct content-types, all 3 families load, no CSP violations).
- Disclosed previously-undisclosed processors in the privacy policy (live site + repo draft):
  **goQR.me** (reward-voucher QR images) and **Have I Been Pwned** (breach check, k-anonymized).
- Disclosed gym-configurable **outbound integrations** (integration-webhook).
- Fixed stale "location is foreground-only / we don't track in background" wording in the repo
  draft to match reality and the live policy.
- Updated `PLAY_DECLARATIONS.md` (Health Connect table, ACTIVITY_RECOGNITION removed) and
  `APP_REVIEW_NOTES.md` (age/minors model + encryption rationale).

### Security
- **`0599_social_posts_select_gym_scope.sql`** — scopes the `social-posts` storage SELECT policy
  to owner / same-gym / accepted-friend. Previously any authenticated user could sign or enumerate
  **every** member's social-feed photo across all gyms. ⚠️ Test on staging before prod (see file header).

### Decisions (no code change needed)
- **Encryption export:** keep `ITSAppUsesNonExemptEncryption=false` + documented standard-crypto
  exemption (your choice). In App Store Connect, answer that the app uses standard encryption
  exempt from export documentation.
- **Age gate:** keep the gym-vouching model + App Review note (your choice). 13+ (16+ EU) by
  self-attestation; gym confirms eligibility at enrollment.
- **`check_ins.latitude/longitude`:** left in place — the Wallet-pass generator reads them
  (null-safe), and nothing ever writes them, so no location is actually collected. Dropping them
  would touch the pass function for zero compliance benefit.
